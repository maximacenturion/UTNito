# Paso 1 - Estructura semántica del login

## Español

### Objetivo
Crear la estructura base del login usando etiquetas semánticas y entender por qué se organiza así.

### Archivos a copiar
- `login.html`

Copiar desde `steps/step-1` hacia `c02-html-start`.

### Conceptos de este paso
- `main`: representa el contenido principal de la página.
- `section`: agrupa un bloque funcional (en este caso, la tarjeta de login).
- `header`: parte superior de la sección (logo + título).
- `footer`: parte final de la sección (texto institucional).

Esta organización mejora legibilidad y facilita luego aplicar estilos por bloques.

### Validación
1. Abrir `c02-html-start/login.html` en el navegador.
2. Verificar que existen `main`, `section`, `header`, `footer`.
3. Verificar que todavía no hay formulario completo.

## English

### Objective
Create the base login structure using semantic tags and understand why it is organized this way.

### Files to copy
- `login.html`

Copy from `steps/step-1` into `c02-html-start`.

### Concepts in this step
- `main`: represents the main page content.
- `section`: groups a functional block (the login card).
- `header`: top section part (logo + title).
- `footer`: bottom section part (institutional text).

This structure improves readability and makes styling by blocks easier.

### Validation
1. Open `c02-html-start/login.html` in the browser.
2. Verify `main`, `section`, `header`, and `footer` are present.
3. Verify the full form is not added yet.
